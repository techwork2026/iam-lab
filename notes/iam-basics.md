# IAM Basics

## What is IAM?

Identity and Access Management (IAM) is the process of managing digital identities and controlling user access to systems, applications, and data.

IAM ensures that the right users have the right access at the right time.

---

## Identity vs Access

- **Identity** = who the user is  
  (Example: John Employee, Contractor, Intern)

- **Access** = what the user can do  
  (Example: view data, reset passwords, manage users)

---

## Core IAM Concepts

### 1. Authentication
Verifying a user’s identity.

Examples:
- Username and password
- Multi-Factor Authentication (MFA)

---

### 2. Authorization
Determining what a user is allowed to do after authentication.

Examples:
- Access to applications
- Permissions within systems

---

### 3. Role-Based Access Control (RBAC)
Access is assigned based on roles instead of individuals.

Example:
- IT Admin → full access
- HR User → limited access

---

### 4. Principle of Least Privilege
Users should only have the minimum access needed to perform their job.

This reduces security risks.

## Key Takeaway

IAM is a critical component of security and ensures proper access control across systems and environments.
IAM is not just about creating users—it involves:

- Managing identities  
- Controlling access  
- Troubleshooting issues  
- Investigating security events  

These skills are essential for IAM and IT support roles.
